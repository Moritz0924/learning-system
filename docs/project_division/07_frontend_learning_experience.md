# 07 前端学习体验分工书

> 返回：[项目分工索引](00_project_division_index.md)  
> 来源：[总架构文档](../../adaptive_private_tutor_v1_architecture.md)

## 模块目标

实现学习者可用的前端体验：完成入学诊断、查看今日任务、与讲师交互、完成测验、查看进度和计划调整差异。

## V1 必须完成

- 入学诊断页：目标、时间、基础测验、偏好、个性化起点提交。
- 今日学习页：今日任务、进度、预计时长、开始学习。
- 讲师页：对话、资料引用、练习、追问。
- 测验页：日测、周测、阶段测验、反馈与错因。
- 学习路径页：阶段路线、知识图谱、未来任务。
- 进度页：掌握度、完成率、复习队列、计划调整原因、调整前后差异。
- 设置页：学习时间、资料管理、长期记忆查看与删除。

## 明确不负责

- 不计算掌握度。
- 不直接调用模型。
- 不直接拼装底层数据库表。
- 不决定计划是否调整。
- 不解析上传文档，只负责上传和展示处理状态。

## 上游输入

- `GET /api/state/current` 返回的状态快照。
- 今日任务、测验、讲师回答和引用。
- `plan_adjustments` 的差异摘要和理由。
- RAG 文档引用标签。

## 下游输出

- 入学诊断提交。
- 学习会话和任务完成提交。
- 用户提问和测验答案。
- 手动调整计划请求。
- 文件上传请求。

## 对接接口/数据表/状态字段

| 类型 | 名称 | 对接说明 |
|---|---|---|
| API | `GET /api/state/current` | 页面初始化主要数据源 |
| API | `POST /api/onboarding/diagnosis` | 入学诊断提交 |
| API | `POST /api/assessments/phase` | 阶段测验创建或更新 |
| API | `POST /api/plans/replan` | 手动触发重排 |
| 状态字段 | `current_state` | 今日任务、复习队列、风险、下一步动作 |
| 状态字段 | `mastery_summary` | 进度页展示 |
| 表/接口 | `plan_adjustments.change_summary`、`rationale_json` | 调整差异展示 |

## 边界规则

1. 前端只展示状态、提交用户操作和展示差异。
2. 前端不直接调用 LLM Gateway 或 MCP。
3. 前端不根据页面数据自行计算掌握度。
4. 计划调整差异必须来自后端返回的 `plan_adjustments` 数据。
5. 所有引用必须展示来源，不展示无法追溯的资料。

## 交付物

- 页面信息架构。
- API 对接清单。
- 状态快照展示映射。
- 计划调整差异展示规则。
- 测验和讲师页面交互流程。

## 验收标准

- 用户能完成入学诊断并看到生成后的当前学习状态。
- 用户能查看今日任务并进入讲师页学习。
- 讲师回答能展示引用来源。
- 用户能完成测验并看到反馈。
- 进度页能展示掌握度变化和计划调整前后差异。
